# Expo Development Client Setup Guide

This project has been configured to use **Expo Development Client** (dev client), which allows you to use custom native code alongside Expo's managed workflow.

## What is Expo Dev Client?

Expo Development Client lets you run your Expo project in development mode with support for custom native code. Instead of using the standard Expo Go app, you build a custom development client that includes your specific native dependencies.

### 2. Configuration Files

- **preview**: For internal testing/distribution

Updated scripts:

- `clean`: Removes node_modules and native directories

### 3. `.gitignore`
Updated to exclude:
- `android/` and `ios/` directories (generated by prebuild)

## Getting Started

### First Time Setup

1. **Install dependencies:**
   ```bash
   npm install
   ```

2. **Generate native code:**
   ```bash
   npm run prebuild
   ```
   This creates the `android/` and `ios/` directories with your custom native code.

3. **Run on iOS:**
   ```bash
   npm run ios
   ```

4. **Run on Android:**
   ```bash
   npm run android
   ```

### Development Workflow

1. **Start the development server:**
   ```bash
   npm start
   ```
   This starts Expo with the dev client flag.

2. **Make changes to your JavaScript/TypeScript code** - they will hot reload automatically.

3. **If you change native dependencies** (add/remove native modules):
   ```bash
   npm run clean
   npm install
   npm run prebuild
   npm run ios  # or npm run android
   ```

## Building for Distribution

### Using EAS Build (Recommended)

1. **Install EAS CLI:**
   ```bash
   npm install -g eas-cli
   ```

2. **Login to EAS:**
   ```bash
   eas login
   ```

3. **Configure your project:**
   ```bash
   eas build:configure
   ```

4. **Build for development:**
   ```bash
   eas build --profile development --platform ios
   eas build --profile development --platform android
   ```

5. **Build for production:**
   ```bash
   eas build --profile production --platform ios
   eas build --profile production --platform android
   ```

### Local Builds

For local builds without EAS:

```bash
# iOS
cd ios && pod install && cd ..
npm run ios

# Android
npm run android
```

## Important Notes

1. **Runtime Version**: The `runtimeVersion` is set to `appVersion` policy, meaning it follows the version in `app.json`. Update this when making native changes that require a new binary.

2. **Updates URL**: Replace `YOUR_PROJECT_ID` in `app.json` with your actual Expo project ID after creating a project on expo.dev.

3. **Native Code Changes**: Any time you add or modify native dependencies, you must run `prebuild` again and rebuild the dev client.

4. **Expo Go vs Dev Client**: This project cannot run in Expo Go anymore because it uses custom native code. You must use the development client.

## Troubleshooting

### Common Issues

1. **"Module not found" errors after adding a package:**
   - Run `npm run prebuild` and rebuild the app

2. **iOS build fails:**
   - Run `cd ios && pod install && cd ..`
   - Clean build folder in Xcode

3. **Android build fails:**
   - Run `./gradlew clean` in the android directory
   - Invalidate caches in Android Studio

4. **Port already in use:**
   ```bash
   npx kill-port 8081
   ```
